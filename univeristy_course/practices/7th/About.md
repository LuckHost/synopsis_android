## 7 практическая

**Кратко:**
Переход на Jetpack Compose, отображение состояния из ViewModel и финальная сборка приложения.

Суть задания:

### 1 — Зачем переходить на Jetpack Compose

К этому моменту приложение уже имеет:

* Repository
* кеширование и оффлайн-режим
* MVVM и ViewModel
* DI через Koin

UI на XML свою задачу выполнил. Теперь необходимо перейти на Jetpack Compose и переписать интерфейс, не меняя архитектуру приложения.

Цель практики — показать, что при правильно выстроенной архитектуре UI можно заменить почти без боли. Если боль есть — значит, где-то раньше была допущена ошибка.

### 2 — Подготовка проекта

В проекте необходимо включить поддержку Jetpack Compose и удалить XML-разметку основного экрана. Activity остаётся точкой входа, но теперь она только вызывает Composable-функцию и передаёт в неё состояние.

ViewModel, Repository и Koin **не переписываются**. Меняется только способ отображения данных.

### 3 — Composable-экран

Необходимо создать основной экран в виде Composable-функции.

Composable:

* получает состояние экрана из ViewModel
* отображает список данных
* показывает состояние загрузки и ошибки
* реагирует на действия пользователя

UI должен быть полностью декларативным. Никаких `findViewById`, никаких адаптеров и ручных обновлений.

### 4 — Список данных

Для отображения списка необходимо использовать LazyColumn.

Каждый элемент списка отображается через отдельную Composable-функцию. Внешний вид может быть простым, главное — читаемым и аккуратным.

### 5 — Связь с ViewModel

ViewModel продолжает хранить всё состояние экрана.

Composable:

* подписывается на состояние
* автоматически перерисовывается при его изменении

Composable **не содержит логики загрузки данных** и не знает, откуда они приходят.

### 6 — Обработка состояний

Экран должен корректно отображать:

* загрузку
* данные
* ошибку

Смена состояний должна происходить автоматически, без ручного обновления UI.

### 7 — Финальная проверка архитектуры

На этом этапе важно убедиться, что:

* UI не знает о Repository
* ViewModel не знает о UI
* все зависимости подключены через Koin
* приложение стабильно переживает пересоздание Activity

Это финальная проверка того, что вся цепочка выстроена правильно.

### Критерии приёмки

* UI полностью реализован на Jetpack Compose
* Используется LazyColumn
* Состояние берётся из ViewModel
* Composable не содержит бизнес-логики
* Архитектура MVVM сохранена
* Проект корректно работает без XML
