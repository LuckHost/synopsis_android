## Что такое Context?

Контекст - абстрактный класс, который предоставляет доступ к глобальной информации о приложении. 
Используется для: 
- получения ресурсов, 
- общения с сетью, файловой системой и БД
- запуска компонентов 
- получения системных компонентов (inflator)

### Виды Context

Сам `Context` - абстрактный класс с одной реализацией и множеством делегатов.

```
Context (абстрактный)
  └── ContextImpl (реализация, единственная)
  └── ContextWrapper (делегат)
    ├── Application
    ├── Service
    └── ContextThemeWrapper
      └── Activity
```

#### ContextWrapper

`ContextWrapper` - обертка, которая делегирует все методы базовому контексту, который получает на вход

```kotlin
// Упрощённая реализация ContextWrapper
class ContextWrapper(base: Context) : Context() {
    private var mBase: Context = base
    
    override fun startActivity(intent: Intent) {
        // Делегируем вызов "базовому" контексту
        mBase.startActivity(intent)
    }
    
    override fun getResources(): Resources {
        return mBase.getResources()
    }
    
    // ... все остальные методы просто делегируют в mBase
}
```

Такой финт ушами позволяет получить 3 преимущества
1. Основной, базовый `Context` создается самим Android и просто при создании инжектится в наследников
2. Базовый контекст можно менять в рантайме (при смене языка/темы)
3. `ContextImpl` создается один раз и просто передается везде, где нужно. Это достаточно большой класс с множеством информации, так что много его наследников бы сильно нагружало систему

#### Наследники ContextWrapper

`Application`, `Service` и `Activity`, являются расширением `Context`


Каждый из них имеет свой жизненный цикл 

|Тип|что такое|время жизни|использование|
|-|-|-|-|
|Application|Контекст всего приложения|Пока живо приложение|, синглтоны, глобальные настройки, библиотеки|
|Service|контекст фоновых задач|Пока сервис не остановлен|Фоновые задачи, не связанные с UI||
|ContextThemeWrapper|Оболочка контекста с темой|Нет собственного времени жизни|Для инфлейта с кастомной темой|
|Activity|Контекст экрана|Пока жива Activity|UI операции, диалоги, старт новых activity|
